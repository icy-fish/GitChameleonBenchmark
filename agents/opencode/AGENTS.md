# OpenCode Benchmark Setup

This directory contains the Docker image and agent-specific guidance for running GitChameleon with OpenCode.

## Goal

Configure OpenCode so it can:

- consume the benchmark prompt generated by the harness
- run with an explicit `provider/model`
- launch reliably inside the benchmark runner

## Prompt Handling

User of this project may specify their own prompt with input. If it's not specified by user, always using following prompt:

```text
Solve file sample_{i}.py in this folder then subsequently save your solutions as result_{i}.py files with the same folder that just completes the starting code provided in the sample and uses the instructions written in the comments at the start of each file.
Remember to always verify output with LSP. Do not run the python file in this workspace.
```

The prompt can be used as string argument for command `opencode run`.

## Model Selection

OpenCode accepts models with `--model provider/model`. If model is not specified by user, using `opencode/minimax-m2.5-free` by default.

Examples:

- `opencode/minimax-m2.5-free`
- `opencode/mimo-v2-pro-free`

Use `opencode models` to inspect models available in the target environment:

```bash
opencode models
```

Use `--variant` only when the selected provider supports it and you need a specific reasoning tier.

## Launch Commands

Build the OpenCode container image:

```bash
docker build -t gitchameleon-opencode:1.1 agents/opencode
```

Run one task through CLI:

```bash
opencode run -m opencode/minimax-m2.5-free --thinking '{prompt}'
```

## Output Expectations

Keep OpenCode output strict:

- Generated code result written into Python file `result_{i}.py`
- Console output represents agent session log, which would be useful for understanding agent's behaviour
